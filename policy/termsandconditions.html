My Personal Health Record

Web Development Capstone Project of Steven Hodges

About this Project

This project idea was developed out of a personal necessity. As an individual who has multiple chronic illnesses, I find it an extreme displeasure trying to manage my health. There is so much data that is generated on a daily basis that it is difficult to keep up with. Worse is the method in which your healthcare providers expect you to keep up with it. Record it on a paper form. Even though many providers may require the same information (e.g. blood glucose or blood pressure) they each have their own forms. Even if the possibility of a digital app is available, no two providers use the same app or share data with each other, even in the same healthcare system.

My project aims to help bring multiple areas of chronic illnesses into one application. In doing this, I hope to simplify how a patient can manage their data. From Appointments to Prescriptions, everything will be in one application. The user will have the option to enter their information and save it. The forms are intuitive and simple to use. The application can generate reports, charts and graphs as well as simple things like calendars or contact list.

Installation Instructions

If you download the repository from GitHub to run on your own, you will need to follow these instructions.

  1. It is presumed that you are a user who is capable of these instructions and have a know-how of how coding and web-development works. Therefore, I will only list the necessary steps to get the project up and running. Please use the web if you need further details on how to install anything other than my code.
  2. You will need a webserver that is capable of serving html, php, css and js files as well as handling MySQL databases. If you are a Microsoft Windows user, two great available options are XAMP and WINAMP. Both are easy to install and setup.
  3. You will need to obtain the MySQL database file for this project. It is included in the repository in the assests-needed.7z file. Alternatively, you can download a copy of the database from my projects folder here. You will need a program like 7zip to extract the file.
  4. Once you have obtained a copy of the database, you will need to import the SQL file into whatever MySQL service you are using.
  5. In your MySQL service, you will need to create a username of "app" (case-sensitive) and that user's password of "AppUser23!" (case-sensitive). That user will need global permissions.
  6. After you have set up the database portion, you will need to complete the web portion. To do this, create a folder in your public html folder of your web server called "myphr". Do not name this folder anything else.
  7. Copy all files of the project into the above folder.
  8. If your web service is correctly running, and you have correctly followed these steps, you should be able to launch the "index.html" page in the root folder to begin the application.
  9. When you have reached the login page, you can use the "Fake Login" link to login to the site. At the time the project is due, I will not have had the capability to include an actual login system.

Extra Features

Extra Feature #1
  • Use arrays, objects, sets or maps to store and retrieve information that is displayed in your app.
  • This feature is implemented in forms.
  • For forms that contain multi-parts, the function stores to the "local storage" and retrieves from "local storage" the data that has been saved.
  • Since uniformity of the data is an important aspect of the website, using data saved to the "local storage" ensures that the user cannot create additional sets of data that are repetitive.
Extra Feature #2
  • Calculate and display data based on an external factor (ex: get the current date, and display how many days remaining until some event).
  • When creating a new patient, it is necessary to establish a number specific to that patient. This number, the Medical Record Number (MRN) will be used to identify the patient across the website as well as associate their information to their account.
  • The function to establish this number combines externally called data (the Year) and data unique to the patient (Name, Birthdate, SSN).
  • Once the patient's "Last Name," "Birthdate," and "SSN" are entered, the function calls for the current year, the "SSN," the first letter of the "Last Name," and the Birthdate (stripped of any additional characters).
  • That number is then assembled into the following format YYYY-SSN-L-Birthdate.
  • For example, patient John Doe Smith, with the SSN of 1234 and the birthdate of 01/01/1995 and entered this month, would have a MRN of "2023-1234-S-19950101"
Extra Feature #3
  • Interact with a database to store and retrieve information (e.g. MySQL, MongoDB, etc).
  • The backend of the project connects to a MySQL database that utilizes PHP 8.1 for connectivity.
  • All form data is stored to the database and retrieved when called upon.
  • Basic CRUD operations exist for most areas.
  • Options that can be used across the website (i.e. gender, race, provider, organization,etc.) are stored in the database. A PHP script calls them and formats them into a HTML '<select>' tag, with the appropriate options.
  • This again stresses the use of uniformed data.
Extra Feature #4
  • Implement modern interactive UI features (e.g. table/data sorting, autocomplete, drag-and-drop, calendar-date-picker, etc).
  • The website uses many different interactive UI features, such as data sorting, autocomplete and calendar-date-picker.
  • The majority of these features are incorporated into forms.
