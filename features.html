Features Under Development

Development

Because this website is a capstone project, I first decided to focus on the requirements that I needed to have in order to complete the project. While I have tried to develop those features first, even they may undergo future changes as my skills improve. Below is a list of features that are visually present only. This page will be linked to from other pages to describe how a feature is intended to function. That way, if a user clicks on a feature, they are not left uninformed.

The Overall Website

The overall website will have a redevelopment. I plan to incorporate the latest versions of Bootstrap, jQuery, Charts.js, etc. as the framework. This will provide a uniformity necessary to host the site, provide easier troubleshooting assistance, open source availability and most importantly make up for developer weakness.

I plan to continue developing my own uniform CSS sheets for many areas. This will be my ability to brand the website as well as show off what skills and knowledge I have come to possess.

The About Page

The "About" page will undergo a redevelopment after the project is graded. References to grading or project requirements will be removed. The page will then focus on three areas:

  • About the Project's History.
  • How to Run the Project.
  • How to use the project.

Policy Pages

Written policy will occur at some point in the future. This will replace the text that is being used as a placeholder currently.

The "Log In" System

Considering that using PHP is better for developing a user back-haul, and I am learning the front end, I have decided to use the open source PHP management system, UserSpice to manage security of PHP pages and the user system. I have become more familiar with this framework and have obtained my developer account for its use. Since it is open sourced, there will not be any issues with the inclusion of it into the website.

Accounts

The utilization of the UserSpice Framework will incorporate into the website a back-end to help users with account functions, such as , resetting passwords, changing usernames, generating support tickets, etc.

Search Function

With the ability to index all pages into one database table, a script to search for features can then be generated to display results based on user search.

Account Menu

This drop-down button will feature links for the user to:

  • Manage their account.
  • Manage their security settings.
  • Change certain site settings.
  • Review Notices.
  • Check system messages.
