My Personal Health Record

Web Development Capstone Project of Steven Hodges

About this Project

This project idea was developed out of a personal necessity. As an individual who has multiple chronic illnesses, I find it an extreme displeasure trying to manage my health. There is so much data that is generated on a daily basis that it is difficult to keep up with. Worse is the method in which your healthcare providers expect you to keep up with it. Record it on a paper form. Even though many providers may require the same information (e.g. blood glucose or blood pressure) they each have their own forms. Even if the possibility of a digital app is available, no two providers use the same app or share data with each other, even in the same healthcare system.

My project aims to help bring multiple areas of chronic illnesses into one application. In doing this, I hope to simplify how a patient can manage their data. From Appointments to Prescriptions, everything will be in one application. The user will have the option to enter their information and save it. The forms are intuitive and simple to use. The application can generate reports, charts and graphs as well as simple things like calendars or contact list.

Project Requirements

The requirements incorporated into this project are:

  1. Use a regular expression to validate user input and either prevent the invalid input or inform the user about it (in all cases prevent invalid input from being stored or saved).
    Forms that have required fields utilize a script to alert the user if a required field is empty or invalid and will prevent the submission of the form until they have been corrected.
  2. Create a function that accepts two or more input parameters and returns a value that is calculated or determined by the inputs. Basic math functions donâ€™t count (e.g. addition, etc).
    Two forms utilize a script that concatenates the name fields (first, middle and last) into the formatted "Display Name" field. The script takes the input and formats it into "Last Name, First Name Middle Name" with the appropriate spacing and punctuation. For the provider it also adds the degree field (i.e. MD, RN, etc).
  3. Visualize data in a user-friendly way. (e.g. graph, chart, etc)
    Utilizing jsCalendar, a current and interactive calendar is displayed. (The future plan is to utilize a php script to activate the calendar with days when entries are (i.e. a date with an appointment will be highlighted and clickable.)
  4. Retrieve data from a third-party API and use it to display something within your app.
    The COVID-19 Portal uses an API to pull current COVID-19 statistics.
  5. Interact with a database to store and retrieve information (e.g. MySQL, MongoDB, etc).
    The Contact Us form saves to the database. The Patient Data and Provider Data utilize a PHP CRUD (Create, Read, Update and Delete) functions. All database functions are in MySQL.
